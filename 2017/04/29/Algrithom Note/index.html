1.1.6.3 方法的性质

  • 方法的参数按值传递。值传递意味着数组参数将是原数组的别名——方法中的参数b变量能够引用调用者的数组并改变其内容（只是不能改变原数组变量本身）
  • 方法名可以被重载。
  • 方法只能返回一个值，但可以有多条返回语句
  • 方法可以产生副作用。返回值为void

    1.1.6.4 递归

    三点注意事项：
  • 递归总有最简单的情况，第一条语句一般是包含return的条件判断语句
  • 递归总是在尝试解决一个规模更小的问题，这样递归才能收敛到最小的情况
  • 递归调用的父问题和尝试解决的子问题之间不应该有交集

1.2 数据抽象

数据类型：一组值和对值进行的操作的集合

Java 主要使用 class 关键字构造引用类型。

对象：保存了某个数据类型的值的实体

抽象数据类型：能够对使用者隐藏数据，支持封装

1.2.1使用ADT

抽象数据类型的定义与静态方法库的不同：

  • ADT中的定义包含构造函数
  • 实例方法不需要static 修饰，目的就是操作数据类型中的值。
  • 某些实例方法是继承而来

1.2.1.2 继承的方法

继承Object基类的方法

  • toString()
  • equals()
  • hashCode()
  • compareTo()

1.2.1.4 对象

对象：能够承载数据类型值的实体

对象的三大特性：

  • 状态：数据类型的值
  • 标识：区分另一个对象，可以认为就是对象在内存中的位置
  • 行为：数据类型的操作

    引用是访问对象的一种方式。

1.2.1.5 创建对象

使用关键字 new 创建

每当用例调用new，系统操作：

  • 为新的对象分配内存空间
  • 调用构造函数初始化对象
  • 返回对象的一个引用

    1.2.1.6 调用实例方法

  • 静态方法：主要作用是实现函数；通过类名直接调用
  • 实例方法：实现对数据类型的操作；通过对象名调用

    1.2.1.7 使用对象

    1.2.1.8 赋值语句

    使用引用类型的赋值语句将创建该引用的一个副本。并不会创建新对象。两个引用指向同一对象

    1.2.1.9 将对象作为参数

    Java 的调用是按值传递，传递参数值的副本

传递对象作为参数时，传递的是对象的引用。方法无法改变原始的引用，但是可以改变对象的值

1.2.1.10 将对象作为返回值

  • 可以返回参数对象，也可以创建一个对象并返回其引用

    1.2.1.11 数组也是对象

  • 一切非原始数据类型的值都是对象

    1.2.1.12 对象的数组

    创建步骤：
  1. 使用方括号语法调用数组的构造函数创建数组对象
  2. 对于数组的每一个元素调用其构造函数创建对象

1.3.1.1 泛型

可以使用集合类的抽象数据类型存储任意类型的数据

中，Item为一个类型参数，是象征性的占位符，可以表示任意数据类型

泛型可以保证我们用一份API处理所有类型的数据

1.3.1.2 自动装箱

类型参数Item必须被实例化为引用类型。

原始数据类型分别对应着它们的引用类型。

Java会自动在引用类型和对应的原始数据类型之间进行转换。

1.3.1.3 可迭代的集合类型

用例需要以某种方式处理集合中的每一个元素，即迭代访问集合中的所有元素

1.3.1.4 背包

不支持删除操作

迭代顺序不确定且与用例无关

元素处理顺序不重要

1.3.1.5 队列

先进先出（FIFO）策略

应用场景：用集合保存元素同时保持它们的相对顺序

1.3.1.6 下压栈

后进先出（LIFO）策略

应用场景：用集合保存元素同时颠倒它们的相对顺序
